    Node Weekly #417

    2022/2/5

    GitHub 推出适用于 npm Registry 的强制 2FA 和更多功能 (opens new window) — 近年来，许多公共代码包注册中心都遇到了用户安全问题，尤其是单个受损包可能会对 npm 生态系统造成严重破坏。因此，GitHub 正在推进 “增强登录验证” 计划，包括在 2022 年初对高影响力软件包的发布者强制执行 2FA。如果你想检查你的 npm 帐户并提高其安全性，现在是时候了！

    为什么你应该检查你的 Node Dependencies (opens new window) — 我目前在 Google 的 Chrome DevTools 团队工作，我们将 node_modules 文件夹检查到源代码管理中。起初这让我觉得很不寻常，但我开始相信这种方法有一些主要的好处，我认为更多的人应该考虑。

    基础即代码的 DevSecGuide (opens new window) — 了解有关 IaC 安全状态的研究、构建 DevSecOps 文化的实际步骤，以及在整个 DevOps 生命周期中嵌入自动化安全的技巧。

    # 简介

    • npm 8.2.0 (opens new window) 已发布。
    • GitHub 正在对其 代码搜索功能 (opens new window) 进行一些重大改进。
    • Chalk 5.0 (opens new window)，流行的终端样式库，现已推出，并已成为纯 ESM。然而 (opens new window)，TypeScript 用户会希望在 TypeScript 4.6 之前坚持使用 Chalk 4。

    你需要了解的 Node.js 内存限制 Node.js 在内存使用方面非常高效，所以你可能永远不会达到内存最大使用限制，但 Node 的内存管理的工作原理以及当程序出现内存限制和内存泄漏相关的问题时应该处理，都值得我们去了解。

    用 Node 对抗 TLS 指纹 (opens new window) — 出于各种原因，许多在线服务会使用指纹来跟踪人们对 Web 的使用 ，TLS 请求的可以做到这一点。

    想要方便的转移域名么? 👀 来看看这个吧 👉 (opens new window)

    使用 Postgres 和 TypeORM 创建视图 (opens new window) — TypeORM (opens new window) 是一种流行的 ORM，它支持 Active Record 和 Data Mapper 模式，可以在 Node.js 中使用。

    如何使用 React、Express.js 和 esbuild 设置服务器端渲染 (SSR) (opens new window)

    # 🛠  代码 & 工具

    graphql-request：用于 Node 和浏览器的最小 GraphQL 客户端 (opens new window) — 可以构建小型脚本和简单的应用程序，而不是像 Apollo 这样的大型项目。

    CSSO 5.0：具有结构优化的 CSS Minifier (opens new window) — 一个清理、压缩和重组 CSS 的工具。新发布的 v5.0 也增加了 ES 模块支持和 CSS Selectors Level 4 支持。

    node-datachannel：Node.js 的 libdatachannel 绑定 (opens new window) — libdatachannel 是各种 WebRTC 标准以及 WebSockets 的基于 C++17 的独立实现，可在 POSIX 平台上使用。

    Node 视频：只需两次 API 调用即可构建视频 (opens new window)

    Nock：HTTP 服务端模拟和封装库 (opens new window) — 假设您正在创建一个使用 HTTP 访问第三方服务的客户端库。 Nock 会让你通过模拟请求/响应来单独测试它。

    Ink 3.2：基于 React 构建的交互式 CLI 应用程序 (opens new window) — 使用 React 风格的组件构建您的命令行应用程序。

    github-unstar：取消星标所有已加星标的 GitHub 存储库 (opens new window) — 如果您在 GitHub 上为太多的 repos 加注星标并且想从头开始，这个快速脚本可能会有所帮助。

    async-sema：用于 async 和 await 的 Semaphore 实现 (opens new window)

    jsdom 19.0：用于 Node 的 Web 标准的纯 JS 实现 (opens new window)

    Multer：处理 “multipart/form-data” 提交的中间件 (opens new window)

    Zod：静态类型推理的 TypeScript-First 模式验证 (opens new window)
