    Node Weekly #430

    2022/3/25

    (opens new window)

    Tao of Node: On Design, Architecture and Best Practices (opens new window) — A developer has boiled down all of the hard-earned best practices and gut feelings he’s picked up for developing high-quality Node apps over time. Opinionated, yes, but high level enough to provide value to most.

    Alex Kondov

    Node v17.8.0 (Current) Released (opens new window) — A relatively minor update: npm upgraded to v8.5.5, an update to the Undici (opens new window) HTTP/1.1 client, and you can now trace requests made by http (opens new window) using perf_hooks to accurately measure round-trip time, etc.

    Bryan English

    (opens new window)

    React Authentication, Simplified (opens new window) — In this article, we lay out a new approach to authentication (plus access control & SSO) in React applications.

    Userfront sponsor

    Malicious npm Packages Targeting Azure Developers (opens new window) — No, this isn’t a repeat of last week’s (opens new window) tale of npm supply chain woes, but a discovery of “hundreds of malicious packages designed to steal PII” targeting developers using packages under the @azure scope.

    Polkovnychenko and Menashe (JFrog)

    Node.js Security Releases to Fix High Severity OpenSSL Bug (opens new window) — Last week they were imminent, but now they’re here in the form of Node 12.22.11 (LTS) (opens new window), 14.19.1 (LTS) (opens new window), 16.14.2 (LTS) (opens new window) and 17.7.2 (Current) (opens new window) (though ignore the latter if you go for 17.8.0 above). The fix? An OpenSSL bug (opens new window) that could cause an infinite loop when parsing certain invalid certificates.

    Joe Sepi (Node.js Project)

    AWS Graviton2 vs Apple M1 for Node.js Performance (opens new window) — A casual and personal benchmark, with all that implies, and while the M1 continues to impress for local development, it’s neat to see other production grade ARM-based systems ready to go too.

    Jamie Knight

    How to Manage Sessions using Passport, Redis, and MySQL (opens new window) — A practical walkthrough of creating a simple Express app with user credentials stored in MySQL, sessions in Redis, and Passport tying it all together. A commonly used pattern boiled down to the essentials.

    Clara Ekekenta

    10 Best Practices to Containerize Node.js Web Applications with Docker (opens new window)

    Snyk sponsor

    ▶  What's Really Going on Inside Your node_modules Folder: The Video (opens new window) — Two weeks ago, we featured an extensive written alternative (opens new window) to this half-hour talk that dives into examples of recent supply chain attacks and some concrete steps you can take to protect your team from this emerging threat – but if you prefer talks/videos, this is well worth the time 😃

    Feross Aboukhadijeh

    Authorization in a Microservices World (opens new window) — Thinking through from a simple approach to full-on attribute-based access control. Not Node specific though the final example is Express-based.

    Alexander Lolis

    # 🛠 Code & Tools

    Dum: An npm Script Runner Written in Rust (opens new window) — Continuing a trend of building JavaScript tools in, well, not JavaScript, the curiously named ‘Dum’ aims to replace npm run and npx to shave milliseconds off the task startup time. YMMV..

    EGOIST

    Video to Reels: Automatically Edit Videos to Post on Instagram Reels (opens new window) — Powered by FFmpeg, ImageMagick, and zx. Features include rotate, resize, add a color filter, normalize audio, and more.

    Diego Fernandes

    Metered Video: Developer First Video Calling API/SDK for Web & Mobile (opens new window)

    Metered Video sponsor

    Chrome Extension CLI: CLI for Building Chrome Extensions (opens new window) — Want to get building an extension for Chrome as quickly as possible? This Node-powered tool aims to get you on the right path ASAP.

    Dutiyesh Salunkhe

    Directus: Wrap an SQL Database with a Real-Time GraphQL + REST API (opens new window) — A Node.js-powered open source system that can act as a frontend to Postgres, SQLite, MySQL, Oracle, and other SQL databases, and provide a modern dashboard, client, and both REST and GraphQL APIs. (Note the GPL license.)

    Directus

    Chinese Random Name: Generate Random Chinese Names (opens new window) — There’s surely an npm package for every want or desire.

    Khaidi Chu

    graphql-request 4.2: A Minimal GraphQL Client Library (opens new window)
    Prisma Labs

    HyperExpress: High Performance Node Server Powered by uWebSockets.js (opens new window)
    Kartik
