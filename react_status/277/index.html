    React Status #277

    2022/2/25

    (opens new window)

    Next.js 12.1 Released (opens new window) — Don’t be fooled by the “.1”, this is a big release for the React powered framework. It introduces a beta version of on-demand incremental static regeneration (ISR (opens new window)), now automatically configures Jest for you, and can create smaller, more efficient ‘standalone’ builds of your app for production deployment.

    🏆  JavaScript to Know for React (opens new window) — A golden oldie we’ve seen doing the rounds again this week. It remains a perfect primer of several key JS features though with examples of features that will help you with React development in particular.

    (opens new window)

    React Authentication, Simplified (opens new window) — In this article, we lay out a new approach to authentication (plus access control & SSO) in React applications.

    An Introductory Guide to Concurrent Rendering (opens new window) — The upcoming (opens new window) React 18 will introduce new concurrent rendering (opens new window) features. What will this offer and why will you want to use it?

    Security of React Native Libraries: The Bad, the Worse and the Ugly (opens new window) — A really deep dive into “issues in third party libraries that drastically affect the security of the main application”. These range from official guides which recommend insecure libraries through easily-misused APIs.

    Deploy a React App on GitHub Pages Automatically (opens new window) — Given that you’re likely on GitHub already, why not use GitHub Pages (opens new window) as a means of deploying React apps? As the author notes, the workload is low and the price is certainly right (at zero).

    Build a Responsive Chat App with Emojis, GIF Support, Edit/Delete Functions, and More (opens new window)

    Creating a 'Reading Progress' Bar with React and Tailwind (opens new window) — Communicating reading progress is just one potential application of this concept. With some adaptation and a data store it should also be possible to track reader engagement—that is, Medium (opens new window)-like stats on how much of the material is actually being read.
